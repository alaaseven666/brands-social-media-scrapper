=== TABLE 4 ===
Agent 1: Sheet Parser Agent
Purpose Accepts a raw File object (.xlsx or .xls), parses it using SheetJS, auto-detects which column contains brand names and which contains URLs, and returns a clean JSON array of competitor objects. Inputs File object from browser file input or drag-and-drop No configuration required — fully automatic column detection  Outputs competitors[] — array of { name: string, url: string } columnMap — { nameCol: string, urlCol: string } for debugging parseError — string | null if something went wrong  Logic / Steps 1. Load file using FileReader as ArrayBuffer 2. Pass buffer to XLSX.read() with { type: 'array' } 3. Call XLSX.utils.sheet_to_json() on the first sheet 4. Scan column headers with regex: /name|brand|company/i for name, /url|website|site|link|domain/i for URL 5. Fall back to first column = name, second column = URL if no matches 6. Filter out blank rows (both name and URL empty) 7. Return structured array + column map

=== TABLE 5 ===
Agent 2: Scraper Engine Agent
Purpose The core of the system. For each competitor, it calls the Anthropic API with the web_search tool enabled, prompts the model to find social media links for that brand's website, parses the JSON response, and returns a structured result. This agent runs once per competitor and is designed to be called concurrently with configurable parallelism. Inputs competitor — { name: string, url: string } index — number (for logging) total — number (for logging) Anthropic API accessible (no key needed in artifact context)  Outputs result — { name, url, socials: { Platform: string }, status: 'ok' | 'unreachable' | 'error' | 'no_links' } Appended to shared results state in real time  Logic / Steps 1. Normalize URL: prepend https:// if no protocol present 2. Build prompt: instruct model to fetch the homepage and return ONLY a JSON object of { PlatformName: URL } 3. Call POST /v1/messages with model=claude-sonnet-4-20250514, max_tokens=1000, tools=[{ type: 'web_search_20250305' }] 4. Extract text blocks from response: data.content.filter(b => b.type === 'text').map(b => b.text).join('') 5. Strip ```json fences, parse JSON safely in try/catch 6. On JSON parse failure: set status = 'parse_error', socials = {} 7. On response containing { error: 'unreachable' }: set status = 'unreachable' 8. On fetch() network failure: set status = 'error' 9. On empty socials object: set status = 'no_links' 10. Push result to shared state and trigger UI update 11. Wait 600ms between calls to avoid rate limiting (configurable)

=== TABLE 6 ===
Agent 3: Report Builder Agent
Purpose Consumes the live stream of results from the Scraper Engine and maintains the report UI. It renders each result as a card as soon as it arrives — the user sees results populate in real time. It also maintains a summary stats bar at the top. Inputs results[] — reactive array updated as A2 completes each site status — 'running' | 'done' | 'stopped'  Outputs Live result cards rendered per competitor Summary stats: X of Y scraped, Z total links found, platform coverage breakdown Visual status badges: found (green), no_links (yellow), unreachable (red), error (red)  Logic / Steps 1. Subscribe to results[] state array (React useState + useEffect) 2. For each result, render a card: brand name, website URL, status badge, platform link pills 3. Platform pills are anchor tags linking directly to the found profile URL 4. Maintain a summary bar with: total processed, total links found, % success rate 5. Show platform breakdown: which platforms had the most presence across all competitors 6. All cards visible at once — no pagination (scroll is fine for 100 items)

=== TABLE 7 ===
Agent 4: Export Agent
Purpose Provides two export buttons — CSV and XLSX — that become active once the run completes (or can also export mid-run). Generates the files client-side using SheetJS and triggers a browser download with no server involvement. Inputs results[] — array of completed result objects SheetJS (xlsx) library loaded via CDN  Outputs competitor_socials.csv — downloadable CSV with one row per competitor competitor_socials.xlsx — downloadable Excel workbook with formatted table  Logic / Steps 1. Define fixed platform columns: Facebook, Twitter, Instagram, LinkedIn, TikTok, YouTube, Pinterest, Snapchat, Threads 2. Map each result to a flat row: { Brand, Website, Status, ...platformCols } 3. CSV: join rows with commas, escape quotes, create Blob with type text/csv, trigger download via anchor click 4. XLSX: use XLSX.utils.json_to_sheet() to build sheet, XLSX.utils.book_append_sheet(), XLSX.writeFile() to trigger download 5. File names: competitor_socials_[YYYY-MM-DD].csv / .xlsx 6. Both exports available immediately after run; also allow mid-run export of completed results so far

=== TABLE 8 ===

=== TABLE 10 ===
Error Type | Cause | Handling
File parse error | Corrupt or wrong-format file | Show inline error, reset to idle state
No URL column found | Sheet has unexpected columns | Default to col 2 as URL, warn user in log
Network failure | API unreachable (e.g. offline) | status = 'error', continue to next competitor
Site blocked / bot detect | Cloudflare, Captcha, etc. | API handles this — model returns unreachable
JSON parse failure | Model returned non-JSON text | status = 'parse_error', log the raw response
Empty result | No socials on site | status = 'no_links', show empty card
Rate limiting | Too many rapid API calls | 600ms delay between calls, configurable in constants
User stops run | Stop button clicked | Set abort flag, finish current call, stop loop
