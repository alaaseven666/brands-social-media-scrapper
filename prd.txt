COMPETITOR SOCIAL LINK SCRAPER
Product Requirements Document & Implementation Plan
Designed for Parallel Antigravity Agent Execution
1. Overview & Problem Statement
Our CRO agency conducts competitive research for every new client onboarding. Currently, the research analyst manually visits each competitor website to find social media profiles — a repetitive, time-consuming task that can take several hours for a typical list of 20–50 competitors.
This tool automates that process. Given an Excel sheet of competitors (brand name + website URL), it visits each site, extracts all social media profile links, and produces a consolidated report — both as an in-app table and a downloadable Excel file.
The implementation is broken into discrete, independently-runnable agents so Antigravity can execute them in parallel and maximize throughput.
2. Goals & Success Criteria
2.1 Primary Goals
Reduce competitor social link research from hours to minutes
Handle 20–100 competitors in a single run
Produce a clean, exportable report with zero manual effort
Be robust to failed/blocked sites — never crash the full run
2.2 Success Criteria
3. System Architecture
The tool is a single-page React application (JSX artifact) that runs entirely in the browser. It calls the Anthropic Messages API directly — no backend server required. This keeps deployment trivial and the whole system self-contained.
3.1 Tech Stack
3.2 Parallel Agent Architecture
The implementation is decomposed into 4 independent agents that Antigravity can run simultaneously. Each agent has a clear input, a single responsibility, and a defined output. A coordinator agent merges the outputs at the end.
4. Agent Specifications
Each agent below can be implemented and tested independently. Antigravity should spin up all four in parallel, with A3 and A4 waiting only on A2's result stream (not its full completion).

6. API Prompt Design
The prompt sent to the Anthropic API for each competitor is the most critical part of the system. It must be precise, tightly constrained, and produce machine-parseable output.
6.1 Prompt Template
6.2 API Request Structure
Model: claude-sonnet-4-20250514
max_tokens: 1000 (social link JSON is always <300 tokens)
Tools: [{ type: 'web_search_20250305', name: 'web_search' }]
Messages: single user message with the prompt above
No system prompt needed — instructions are in the user turn
6.3 Response Parsing
Concatenate all text-type content blocks from data.content
Strip ```json and ``` fences using regex
JSON.parse() inside try/catch — on failure, status = 'parse_error'
Validate result is a plain object (typeof === 'object', not null, not array)
Filter values to only include strings starting with 'http'
If result.error exists, mark as unreachable

10. Known Limitations & Constraints
10.1 Technical Constraints
Social links embedded only in JavaScript (not static HTML) may be missed — web_search mitigates this significantly
Sites behind Cloudflare or strict bot protection may return unreachable even with web_search
API rate limiting: 600ms delay between calls is a safe default, but may need tuning for large lists
SheetJS loaded from CDN — requires internet access in the artifact sandbox
All processing is client-side — large sheets (200+ competitors) may be slow in the browser
10.2 Data Constraints
URL column must contain valid URLs or at minimum recognizable domain names
The tool cannot verify if a found social profile is the official brand account vs a fan page
Social links in image elements (e.g. SVG icons with no href) will not be detected
Platforms not in the predefined list (e.g. Bluesky, Mastodon) will not be captured
10.3 Out of Scope for v1
Following inner pages (about, contact) — homepage only
Authenticating to any website
Google Sheets input format — .xlsx only
Scheduling / recurring runs
Historical tracking / change detection between runs

Agent 1: Sheet Parser Agent
Purpose
Accepts a raw File object (.xlsx or .xls), parses it using SheetJS, auto-detects which column contains brand names and which contains URLs, and returns a clean JSON array of competitor objects.
Inputs
File object from browser file input or drag-and-drop
No configuration required — fully automatic column detection

Outputs
competitors[] — array of { name: string, url: string }
columnMap — { nameCol: string, urlCol: string } for debugging
parseError — string | null if something went wrong

Logic / Steps
1. Load file using FileReader as ArrayBuffer
2. Pass buffer to XLSX.read() with { type: 'array' }
3. Call XLSX.utils.sheet_to_json() on the first sheet
4. Scan column headers with regex: /name|brand|company/i for name, /url|website|site|link|domain/i for URL
5. Fall back to first column = name, second column = URL if no matches
6. Filter out blank rows (both name and URL empty)
7. Return structured array + column map
Agent 2: Scraper Engine Agent
Purpose
The core of the system. For each competitor, it calls the Anthropic API with the web_search tool enabled, prompts the model to find social media links for that brand's website, parses the JSON response, and returns a structured result. This agent runs once per competitor and is designed to be called concurrently with configurable parallelism.
Inputs
competitor — { name: string, url: string }
index — number (for logging)
total — number (for logging)
Anthropic API accessible (no key needed in artifact context)

Outputs
result — { name, url, socials: { Platform: string }, status: 'ok' | 'unreachable' | 'error' | 'no_links' }
Appended to shared results state in real time

Logic / Steps
1. Normalize URL: prepend https:// if no protocol present
2. Build prompt: instruct model to fetch the homepage and return ONLY a JSON object of { PlatformName: URL }
3. Call POST /v1/messages with model=claude-sonnet-4-20250514, max_tokens=1000, tools=[{ type: 'web_search_20250305' }]
4. Extract text blocks from response: data.content.filter(b => b.type === 'text').map(b => b.text).join('')
5. Strip ```json fences, parse JSON safely in try/catch
6. On JSON parse failure: set status = 'parse_error', socials = {}
7. On response containing { error: 'unreachable' }: set status = 'unreachable'
8. On fetch() network failure: set status = 'error'
9. On empty socials object: set status = 'no_links'
10. Push result to shared state and trigger UI update
11. Wait 600ms between calls to avoid rate limiting (configurable)
Agent 3: Report Builder Agent
Purpose
Consumes the live stream of results from the Scraper Engine and maintains the report UI. It renders each result as a card as soon as it arrives — the user sees results populate in real time. It also maintains a summary stats bar at the top.
Inputs
results[] — reactive array updated as A2 completes each site
status — 'running' | 'done' | 'stopped'

Outputs
Live result cards rendered per competitor
Summary stats: X of Y scraped, Z total links found, platform coverage breakdown
Visual status badges: found (green), no_links (yellow), unreachable (red), error (red)

Logic / Steps
1. Subscribe to results[] state array (React useState + useEffect)
2. For each result, render a card: brand name, website URL, status badge, platform link pills
3. Platform pills are anchor tags linking directly to the found profile URL
4. Maintain a summary bar with: total processed, total links found, % success rate
5. Show platform breakdown: which platforms had the most presence across all competitors
6. All cards visible at once — no pagination (scroll is fine for 100 items)
Agent 4: Export Agent
Purpose
Provides two export buttons — CSV and XLSX — that become active once the run completes (or can also export mid-run). Generates the files client-side using SheetJS and triggers a browser download with no server involvement.
Inputs
results[] — array of completed result objects
SheetJS (xlsx) library loaded via CDN

Outputs
competitor_socials.csv — downloadable CSV with one row per competitor
competitor_socials.xlsx — downloadable Excel workbook with formatted table

Logic / Steps
1. Define fixed platform columns: Facebook, Twitter, Instagram, LinkedIn, TikTok, YouTube, Pinterest, Snapchat, Threads
2. Map each result to a flat row: { Brand, Website, Status, ...platformCols }
3. CSV: join rows with commas, escape quotes, create Blob with type text/csv, trigger download via anchor click
4. XLSX: use XLSX.utils.json_to_sheet() to build sheet, XLSX.utils.book_append_sheet(), XLSX.writeFile() to trigger download
5. File names: competitor_socials_[YYYY-MM-DD].csv / .xlsx
6. Both exports available immediately after run; also allow mid-run export of completed results so far
